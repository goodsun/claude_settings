# 3人のエキスパートによるPRレビュー

GitHub上のPRを、3人の異なる専門性を持つエンジニアがレビューします。

## Step 1: PR情報の取得

引数 `$1` はPR番号（例: `123`）またはPR URL（例: `https://github.com/owner/repo/pull/123`）のどちらでも指定できます。

まず、PRの情報を取得してください：

```bash
gh pr view $1
```

PRの差分を取得：
```bash
gh pr diff $1
```

## Step 2: 3人のエキスパートによるレビュー

以下の3人のエージェントを**順番に**使用して、それぞれの専門的な観点からレビューを実施してください。

### レビュアー1: ベテランWebエンジニア（15年経験）
veteran-web-engineer エージェントを使用して、**変更容易性と可読性**の観点からレビューしてください。

### レビュアー2: セキュリティエンジニア（10年経験）
security-engineer エージェントを使用して、**セキュリティ脆弱性**の観点からレビューしてください。

### レビュアー3: パフォーマンスエンジニア（15年経験）
performance-engineer エージェントを使用して、**パフォーマンスと負荷対策**の観点からレビューしてください。

## Step 3: 統合レポート

3人のレビュー結果をまとめて、以下の形式で統合レポートを作成してください：

```
# PR #$1 レビュー統合レポート

## サマリー
- PR: #$1
- レビュー対象: [変更ファイル数]ファイル
- 重大な問題: X件
- 改善推奨: Y件

## 最優先で対応すべき項目
1. ...
2. ...

## 各エキスパートからのレビュー

### 可読性・変更容易性（ベテランWebエンジニア）
[レビュー結果の要約]

### セキュリティ（セキュリティエンジニア）
[レビュー結果の要約]

### パフォーマンス（パフォーマンスエンジニア）
[レビュー結果の要約]

## 推奨アクション（優先度順）
1. [Critical] ...
2. [High] ...
3. [Medium] ...
```
